# F1 Race Engineer - Milestone 1

Goal of this milestone: confirm the pipe from F1 25 to a running app works, by
displaying your live current and last lap time in a window.

## 1. In-game setup (F1 25 / F1 25: 2026 Season Pack)

1. Launch the game.
2. Go to Options > Settings > Telemetry Settings.
3. Set **UDP Telemetry** to On.
4. Set **UDP IP Address** to `127.0.0.1` (since the app runs on the same PC as the game).
5. Set **UDP Port** to `20777` (matches the default in this app - change both together if you ever need a different port).
6. Set **UDP Send Rate** to `20Hz` or `60Hz`.
7. Set **UDP Format** to `2025`.
8. Leave **UDP Broadcast Mode** off.
9. Back out of the menu so settings save.

## 2. Running the app

1. Open this folder in VS Code.
2. Open a terminal in VS Code (Terminal > New Terminal).
3. Run:
   ```
   dotnet run
   ```
4. The first run will download the F1Game.UDP package and compile - this can take a minute or two.
5. A window should open titled "F1 Race Engineer - Milestone 1".

## 3. Testing it

1. With the app running, start a Time Trial or Practice session in the game.
2. Click **Connect** in the app.
3. Drive out of the pits and complete a lap.
4. "Current Lap Time" should update live; "Last Lap Time" should populate once you cross the line.

## About the API reference

`docs/F1GameUDP_API_reference.txt` is a full reflection dump of every public
type and member in the exact F1Game.UDP 26.0.0 DLL installed on this machine -
generated by running a small diagnostic console app against the real
compiled library, not from documentation. It's what resolved the earlier
`CS0246` build errors (the types exist under `F1Game.UDP.Packets`,
`F1Game.UDP.Enums`, and `F1Game.UDP.Data` - not the root `F1Game.UDP`
namespace, which is why `using F1Game.UDP;` alone didn't find them).

Keep this file - it's the authoritative source for exact property names when
we build out gaps, tyre age, and alerts in later milestones, instead of
guessing from online docs again.

## If it doesn't compile

This version has been checked against the confirmed API reference above, so
it should build cleanly. If it still doesn't, paste the exact error back.

## If it compiles but shows no data

- Confirm Windows Firewall isn't blocking the app (it may prompt on first run - allow it).
- Confirm the game is actively in a driving session, not a menu or replay.
- Confirm no other telemetry app (SimHub, RaceLab, CrewChief, etc.) is already
  listening on port 20777 - only one app can bind a given port at a time.
